= Erfahren Sie mehr über ONTAP Unterstützung und -Funktionen für Proxmox-Hosts.
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Die für die Hostkonfiguration mit NVMe over Fabrics (NVMe-oF) unterstützten Funktionen variieren je nach Ihrer ONTAP und Proxmox-Version.

[cols="30,20,20"]
|===
| ONTAP Funktion | Proxmox-Hostversion | ONTAP-Version 


| Bietet ONTAP Details für die Namespaces NVMe/FC und NVMe/TCP unter Verwendung der nativen Schnittstelle. `nvme-cli` Paket.  a| 
8.0
| 9.10.1 oder später 


| NVMe/TCP ist eine vollständig unterstützte Unternehmensfunktion.  a| 
9.0
| 9.10.1 oder später 


| NVMe- und SCSI-Datenverkehr werden auf demselben Host unterstützt, wobei NVMe Multipath für NVMe-oF-Namespaces und dm-Multipath für SCSI-LUNs verwendet werden.  a| 
8.0
| 9.4 oder später 
|===
ONTAP unterstützt die folgenden SAN-Hostfunktionen unabhängig von der auf Ihrem System ausgeführten ONTAP Version.

[cols="70,30"]
|===
| Funktion | Proxmox-Hostversion 


| Native NVMe-Multipathing ist standardmäßig aktiviert.  a| 
8.0



| Der `nvme-cli` Das Paket enthält Skripte zur automatischen Verbindung, sodass keine Skripte von Drittanbietern benötigt werden.  a| 
9.0



| Die native udev-Regel in `nvme-cli` Das Paket bietet Round-Robin-Lastverteilung für NVMe-Multipath  a| 
9.0

|===

NOTE: Einzelheiten zu unterstützten Konfigurationen finden Sie imlink:https://mysupport.netapp.com/matrix/["Interoperabilitäts-Matrix-Tool"^] .
